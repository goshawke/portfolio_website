Battle Buddies Android App

In a team of 2, we developed an Android Application called Battle Buddies. Our goal was to create a phone application that assists in the safety of college-aged people who inevitably go to parties, clubs, or bars. We wanted to encourage people to go out in groups (we call these people “Battle Buddies”) and provide them with simple tools for ensuring the safety of one another, while still encouraging our users to enjoy themselves.

Key Tools & Skills: Android Mobile App Development, Wireframing, Prototyping, Java, XML, Firebase Database, & Google API

COVID Dashboard Android App

This project was developed along with another student for a Software Development course. The goal was to develop an Android mobile application to track and monitor risk regarding the COVID-19 virus. Users are able to: log symptoms related to COVID, any medicines currently being taken, doctor visits, travel, news from friends and family regarding COVID diagnoses, and restaurant visits. Finally, users can access a dashboard indicating the likelihood that they were exposed to COVID given all the information collected. The calculation was based on an arbitrary formula and is not scientific.

Key Tools & Skills: Android Mobile App Development, Wireframing, Prototyping, Java, XML

Github Web Scraper

This was a project developed in a small team for a Software Development course. The goal was to develop a desktop application for searching Git projects using both single-keyword search and multiple-filtering. Additionally, capabilities were developed to display the projects on a web UI and, if desired, to show the file structure and dependencies of individual projects. Keyword search was acomplished through API calls. Categorization and Filtering were acheived with the help of an OWL/XML ontology that I personally built.

Key Tools & Skills: Python, Protege Ontology development, Figma Wireframing, API development, Node.js, Web Scraping

Student Research: Convolutional Neural Network for Action Classification

As part of a small team, we developed a Convolutional Neural Network (CNN) for the purpose of video classification. We trained the model on the UCF-101 dataset which features 101 categories of different actions and 13,320 video clips collected from YouTube. We were pleased with the results of the CNN we designed. The model architecture went through many different iterations, with varying amounts of hidden layers and several rounds of hyper-parameter optimization before we settled on our final model. The training accuracy was 99.8% and validation accuracy was 69.5% at the final epoch. While the training accuracy suggests there could be overfitting, the validation accuracy is high enough that we were satisfied with the model’s ability to learn and predict. Our model proved to be as effective, if not more effective, than contemporary models on the same dataset.

Key Tools & Skills: Python, Pandas, Pytorch, Jupyter Lab, Machine Learning
